Есть форма на сайте, которая рассчитывает возможность оформления паспорта РФ в зависимости от возраста (по действующему законодательству). 
Поле принимает на вход целые числа от 0 до 115 включительно. 
В результате получаем сообщение: “Нельзя оформить”, “Можно оформить” или “Ошибка”.

Разбейте входные данные по технике граничных значений и определите результат для каждого случая
Разбейте входные данные на классы эквивалентности, приведите пример входного значения для каждого класса и укажите результат

Классы эквивалентности:
1. Пустое поле - "Ошибка"
2. Нечисловые символы (буквы, спецсимволы, эмоджи, пробелы) - "Ошибка" (Пример: АидGKS5#_^)
3. Нецелые числа - "Ошибка" (Пример: 15,5)
3. Отрицательные числа - "Ошибка" (Пример: -54)
4. [0-13] - "Нельзя оформить" (Пример: 9)
5. [14-115] - "Можно оформить" (Пример: 51)
6. [116-бесконечность) - "Ошибка" (Пример: 986)
7. Можно провести отдельную проверку - ввести только пробелы
*Нечисловые символы могут быть недоступны для ввода, но проверить их все равно нужно

Граничные значения (использую технику two-point):
1. -1 ("Ошибка"), 0 ("Нельзя оформить")
2. 13 ("Нельзя оформить"), 14 ("Можно оформить")
3. 115 ("Можно оформить"), 116 ("Ошибка")
